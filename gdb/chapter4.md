# Chapter4

## 其他汇编指令

```bash
test op1 op2
```
让两个操作数进行AND运算，不存储结果，根据结果内容更新RFLAGS的标志位，主要设置ZF, SF, OF, CF。test指令始终让CF与OF清零，如果后面跟jxx指令就看ZF位。

## 地址计算

$$
有效地址 = 基址+（变址\times比例因子）+OFFSET
$$

### 寻址方式

| 类型 | 汇编示例 | 计算公式 | 说明 |
| :--- | :--- | :--- | :--- |
| 立即数寻址 | `$0x400` | $0x400 $  | 数据直接在指令里，不访问内存。 |
| 寄存器寻址 | `%rax` |  $ R[\text{rax}] $  | 数据在寄存器里，速度最快。 |
| 绝对寻址 | `0x4000` | $0x4000 $  | 直接给出内存地址（Linux 用户态代码很少用）。 |
| 寄存器间接寻址 | `(%rax)` |  $ M[R[\text{rax}]] $  | 寄存器里的值就是地址（类似 C 的 `*p`）。 |
| 基址+偏移 | `8(%rbp)` |  $ R[\text{rbp}] + 8 $  | 访问结构体成员或局部变量常用。 |
| 变址寻址 | `(%rax, %rcx)` |  $ R[\text{rax}] + R[\text{rcx}] $  | 两个寄存器相加。 |
| 比例变址 | `(%rax, %rcx, 4)` |  $ R[\text{rax}] + R[\text{rcx}] \times 4 $  | 数组访问神器。`%rcx` 是索引，`4` 是 `int` 的大小。 |
| 完整模式 | `16(%rax, %rcx, 8)` |  $ R[\text{rax}] + R[\text{rcx}] \times 8 + 16 $  | 最复杂的组合，一步算出地址。 |

LEA（Load Effective Address）计算

```bash
leaq 16(%rax, %rcx, 4), %rdx
```

经常用于算地址和优化加减乘除（效率高只用一个时钟周期），也不会从内存中读数据（不访问内存）。
