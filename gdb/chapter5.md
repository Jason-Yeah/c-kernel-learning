# 优化程序性能

## 内存别名使用

```c
x = 1000; y = 3000;
*p = x;
*q = y;
t1 = *q;
```

如果p和q指向一样，会有覆盖的情况；

## other

- malloc参数只要一个，就是分配的字节数，但不初始化内存，直接从堆中划个空间给用，可能存在脏数据
- calloc两个参数，一个元素个数，一个元素的大小，会把内容初始化为0。

### 内存泄露

分配数据失败要free已分配的头部结构体。如果不free，那之前分配的头data还在占空间，比如

```c
vec_p res = (vec_p) malloc (sizeof(vec_rec));
```

不free，局部变量res从栈消失，被malloc分配的vec_rec还在堆HEAP中，OS默认程序还在使用这个，不会分配给别人，如果一直调用new且频繁失败，“孤儿内存”会越来越多，最后干爆系统资源。（程序关了自然释放，但程序一直运行就难绷了）。
