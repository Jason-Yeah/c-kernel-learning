# C++中别名

&这个就是别名在c++中，但实际上这byd有俩面孔

1. 在表达式中是取地址符号，`p = &a;`这就是C中取地址操作，p最后拿到的是a在内存中的地址。
2. 在类型声明中，`int &b = a;`这是引用声明符

为什么叫别名，比如

```cpp
int a = 10;
int& b = a;
```

在底层a和b指向内存中同一个地方，a变b变，b变a变，物理上a和b一个东西，b是a的另一个名字。

但他妈真相是b在内存中存的是a的地址，只不过c++给了个自动转换的协议

- 写`b = 20`，编译器在底层整成`*b = 20`了
- 用b时，编译器自动帮助解引用

## 示例

对于如下示例

```cpp
char operator[](int i) { return pText[i]； }
```

`pText[i]`是内存中的一个字符比如'H'，函数执行时会把该字符copy一份，放到一个临时寄存器里然后传递出来。

实际最后拿到是一个副本，和本体没一点关系，那你做形如`cctb[0] = 'J';`就一定会报错，因为不能给临时副本赋值。（注意，cctb[i]这里实际上调用的重载的opterator[]，如果返回值定义的是char，那cctb[0]只是个临时副本，是从寄存器里取出来的，没有内存地址）

如果改成

```cpp
char& operator[](int i) { return pText[i]; }
```

返回的是pText[i]的引用，底层看返回的确实是pText[i]的内存地址，后面如果写`cctb[0] = 'J';`就实际上是顺着地址改的，直接改的是本体。

### 为什么不写T*

为什么让对象用起来更像是数组，比如原生数组`arr[0] = 10;`，那C++让vector也用起来差不多`vec[0] = 10;`不要打*号了。

所以有T&这东西，表面是跟之前的变量差不多，底层干指针的活。
